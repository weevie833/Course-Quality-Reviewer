Overview

Now that your sponsor has signed off on the project charter and you have an established scope, you are ready to begin the planning for the project. This starts with refining the specifics of the project and identifying the actual work tasks that need to be accomplished to achieve the project’s outcomes. You will also work with other team members to define, sequence, and estimate activity durations. You will analyze the constraints and dependencies between the tasks.

At the end of this module you should be able to:

  1. Distinguish how a WBS functions as an organizing instrument for a project.
  2. Describe the linkage between the WBS and the project network diagram.
  3. Determine how constraints and dependencies affect the course of a project.
  4. Describe how different methods of diagramming a project network can be used to optimize project schedules.
  5. Apply analogous, parametric, and three-point techniques to produce a project estimate.
  6. Develop a project network diagram.
  7. Analyze a project network to identify the critical path.
  8. Articulate how estimating project times and costs affects project planning and control.
  9. Identify methods, uses, and advantages and disadvantages of top-down and bottom-up estimating methods.
  10. Develop a project schedule.
