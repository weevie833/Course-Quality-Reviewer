Overview

Change requests can be initiated any time after the project starts and can come from the customer, sponsor or the project team. Guiding the project team through the defined plan is the responsibility of the project manager. Once the project comes to an end, the project closure activities must be performed. Even if a project is abandoned, the project manager must ensure that it is closed properly to tie up any loose ends that could cause issues later on.

At the end of this module you should be able to:

  1. Manage integrated change control.
  2. Analyze the impact of change and take appropriate actions.
  3. Describe how project close out activities are critical to business operations.
  4. Define project closure activities.
  5. Determine lessons learned.
