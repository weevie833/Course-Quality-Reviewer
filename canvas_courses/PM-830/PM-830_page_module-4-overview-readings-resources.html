Overview

Key to success as a project manager is the ability to deliver the product(s) desired by the business sponsoring the project as well as those required by certain departments, such as security and operations.  This entails defining the various requirements at the lowest reasonable level of detail, such that the requirements can be allocated to components of the work breakdown structure. We will review the product breakdown structure often used to capture the specifics needed. Requirements traceability through the project’s life cycle model will also be addressed. This traceability, often expressed in a traceability matrix, is needed to support quality control, to ensure all requirements have been addressed and function as the business desires.

Closely associated with requirements management is configuration management. That means all the various components of the system are maintained as a single cohesive package as the project proceeds. These components might consist of operating system, operating system parameters, application software, databases, and documentation, among others. This ensures the final product contains all appropriate software and hardware components when released into operation, and that the release fulfills all requirements identified within the product breakdown structure or requirements traceability matrix.

Module Objectives:

  • Students will be able to develop a requirements management plan for a project, a plan that includes all requirements, such as user, operations, and security requirements.
  • Students will be able to write a configuration management plan detailing how various versions of the project’s products will be controlled.
